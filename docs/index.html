This editor helps you configure your Multi-Report script settings. Make sure you understand what each setting does before changing it. The -config is likely a better option where you just answer questions, however some of the advanced configurations are missed by people. This will expose these settings.

Load: Click this to load settings from a multi_report_config.txt file. This is useful for editing existing configurations.

Save: Click this to save your current settings to a multi_report_config.txt file. This file will be downloaded to your computer, typically your downloads folder.

Section Specific Help: Below is additional help in order to configure the Drive Customization section.

The data provided will identify the specific format the data is to be formatted. If this seems complicated, use the -config option of the Multi-Report script. Ignore Drives List: This will have the script ignore any matching drive serial number. Format: This is a simple comma delimited format. List each drive serial number separated by a comma. Example: (KV5T8G,OU812,TESTME).

The following all have the same format: ATA Errors List, CRC Errors List, MultiZone List, ReAllowcated Sector List, ReAlocated Sector Events List, and Media Errors List. Format is drive serial number, colon, compensation number, comma, repeat. Example: (KV5T8G:6,OU812:1,TESTME:3).

The Drive Locations format is a simple comma delimited format. Example: (KV5T8G:"Tray 1",OU812:"Removable Rack 3B",TESTME:"Died and went to heaven").

The Drive Warranty is nice to have if for example you have a drive with a 3 year warranty, you enter the warranty expiration date and the report tells you how long until the warranty expires. Once you have exceeded the warranty date it counts up and you get to see how many years, months, days your drive has lasted. The format: serial number:YYYY-MM-DD,serial number:YYYY-MM-DD. Example: (KV5T8G:2023-06-15,OU812:2025-09-23,TESTME:2024-03-12).

The most complex format has been saved for last Custom Drive List. You are better to use the -config option built into the script, however here is the format is one continuous line, broken into pieces for your viewing pleasure: 

serial:tempwarn:tempcrit:sectorswarn:sectorscrit:
    reallocwarn:multizonewarn:multizonecrit:rawreadwarn:rawreadcrit:
    seekerrorswarn:seekerrorscrit:testage:testAgeOvrd:heliummin:
    wearleveladj:farmoffset

(testAgeOvrd 0=Default, 1=Ignore, wearleveladj d=Default r=Reverse value n=Normalized i=Ignore, farmoffset d or i)
